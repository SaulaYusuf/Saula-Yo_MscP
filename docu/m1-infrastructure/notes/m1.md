# M1: Infrastructure As-Built Report

## Overview
Successfully provisioned a containerized Hyperledger Fabric 2.5.16 environment. The network is architected to solve the research gap of data silos and throughput bottlenecks by utilizing a multi-channel Master-Slave configuration.

## System Specification
- **Orchestration:** Docker Compose
- **Network Topology:** 2 Org Peer Network + Orderer Org (Raft Consensus)
- **Partitioning Strategy:**
    - `masterchannel`: Dedicated to logistics handovers and audit trails.
    - `slavechannel`: Dedicated to high-frequency IoT environmental telemetry.

## Verification Commands Executed
1. **Network Initialization:** `./network.sh up -ca`
2. **Channel Partitioning:**
   - `./network.sh createChannel -c masterchannel`
   - `./network.sh createChannel -c slavechannel`
